01 · Where it applies

Four places to start.

These are the four workflows that show up most often in the discovery workshop. Each one has been scoped against a real bank or insurer's operating model.

FRAUD90 days

Fraud investigation.

An investigator agent pulls the transaction graph, walks the correspondent and beneficial-owner chain, checks sanctions lists, and prepares the SAR draft. A human reviewer signs every filing. The 14-day backlog drops to same-day in the first quarter.

AML · KYC120 days

AML and KYC reviews.

Periodic KYC refresh, EDD on high-risk segments, and alert disposition for the transaction monitoring system. Auto-clears the noise tier, packages the borderline tier for a human, and escalates true positives with full lineage.

CLAIMS120 days

Claims handling.

For P&C and life carriers. A triage agent reads the FNOL, the policy doc, and the supporting attachments, then routes to fast-pay, adjuster, or SIU. Cycle time on first notice drops, and leakage on auto-pay claims gets visible to actuarial.

ADVISOR90 days

Advisor and banker productivity.

A copilot inside the advisor's existing CRM that drafts client communications, prepares meeting prep packets, and surfaces compliance-aware next-best-actions. Branch staff use the same surface in retail. Every draft is logged.

02 · How we ship this

AI-DLC in financial services.

Four pillars, mapped to the workflows above. Each pillar is the same code on every engagement, with finance-specific contracts and gates layered on top.

// pillar 01 · agents

One job per agent.

The fraud investigator, the AML reviewer, the claims triager, the advisor copilot. Each one has a defined input, a defined output, and a defined override path. No reasoning agent is asked to do three jobs at once.

03 · Built for the audit

How we keep the CISO comfortable.

A bank's CISO and chief risk officer have veto. The substrate is shaped around what they need to sign without flinching.

// 01

Lineage by construction.

Every retrieval, every prompt, every model call, every output gets a signed record. There is no path through the substrate that skips lineage. Audit responses are queries against a lake, measured in minutes.

// 02

Evaluation gates.

Regression suites run on every change. A prompt rewrite that drops accuracy below the gate fails the build. Drift detection on production traffic raises an alert before a regulator notices.

// 03

Override trail.

When a human overrides an agent's recommendation, the override gets the same signed record. The CRO can see the rate, the pattern, and the cases. Override frequency is its own metric in the executive report.

// 04

Regulator-ready exports.

One-click export of a case's full lineage in a format the regulator can consume. SAR-ready, FCA-ready, APRA-ready. The audit team's job stops being a recovery project and starts being a query.
